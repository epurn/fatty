# Mobile Skill

Use for Expo, React Native, iOS UI, navigation, forms, charts, camera, barcode, and local app state.

## Standards

- iOS is the primary UI.
- Use clean native-feeling layouts with compact data density.
- Avoid chatbot UI as the main surface; natural language is input, structured timeline is output.
- Keep text accessible, readable, and nonjudgmental.
- Support loading, estimating, needs-info, failed, and edited states.
- Use icons for evidence/status with accessibility labels.
- Keep camera and barcode flows permission-aware.

## Checks

When mobile tooling exists, run typecheck, lint, unit tests, and screenshot/smoke checks for core screens.
